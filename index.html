Grid Control: Software platform to support the Smartgrid

The current power grid is a dinosaur dating back to late 1800s. It is prone to outages caused by oscillatory disturbances. Outages cost US economy $80 billion every year. It has high distribution losses which costs US economy nearly $20 billion every year. The SmartGrid is equipped with many networked sensors. This sensor data can be used by real time analysis applications to provide better integration of variable renewable energy sources such as wind and solar opportunities for dynamic pricing, demand response load balancing, and many more. Since there are 15000 such sensors distributed all over the US, it would be very difficult to have system running on a single server. So I, in a team of 4, have built a distributed system to collect sensor data from the SmartGrid and apply machine learning techniques to estimate the state of the grid. We have used techniques like sharding and replication to make the system reliable and scalable. We have adopted a model where we have a central management system, which is consistently replicated on multiple machines, which manages the network overlay of the system which makes the system fault tolerant. We reduce the effect spicky nature of TCP, which causes variation in the latencies, by sending sensor data over mulitple streams to multiple nodes. This approach not only reduces the variation in the latencies but also makes the system tolerant to weird network connection failures.

Cloud based Execution Engine for GE MAPS and MARS

The General Electric (GE)'s Energy Consulting group has developed the Concorda Software Suites - MAPS (Multi Area Production Simulation Software) and MARS (Multi Area Reliability Simulation Software). These products are internationally known and widely used for planning and simulating electric power grids, assessing the economic performance of large electricity markets, and evaluating generation reliability. As the size of the systems modeled by the software and the number of sensitivities increase, the need to improve computation time for the analysis engines has become increasingly crucial. To address these issues, we have developed a framework that enables these applications to be easily executed in a cloud computing environment, which provides reliable, fast computation as well as an interface for clients to monitor their job status. Using this framework, the clients can take the advantage of cloud based execution and enjoy a significant computation time reduction.
